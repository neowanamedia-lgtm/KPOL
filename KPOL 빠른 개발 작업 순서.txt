KPOL 빠른 개발 작업 순서

0. 방향 확정
- 서비스 정의:
  “뉴스 데이터를 기반으로 대한민국 정치 흐름을 보여주는 종합 정보 터미널”
- 금지:
  댓글 / 토론 / 사용자 평가 / 좋아요 / 선거 예측 / 정치적 논평
- 허용:
  뉴스 언급량 / 키워드 연결 / 기사 증가율 / 근거 기사 / 기간별 흐름

1. MVP 범위 축소
- 1차 대상:
  국회의원
  주요 정당
  주요 광역단체장
- 1차 데이터:
  뉴스 기사 제목
  기사 날짜
  언론사
  기사 URL
  본문 요약은 후순위
- 1차 지표:
  오늘 기사 수
  최근 7일 기사 수
  전일 대비 변화
  7일 평균 대비 변화
  많이 함께 나온 키워드
  관련 기사 목록

2. 정치인 DB 먼저 만들기
- politicians 테이블
  id
  name
  party
  position
  region
  category
  aliases
  is_active
- 정당명 변경, 직책 변경 가능성 고려
- 동명이인 구분 필수

3. 뉴스 수집 구조 만들기
- 초기에는 완전 자동보다 반자동 추천
- 네이버뉴스/구글뉴스/RSS/공개 검색 API 중 현실적인 방식 선택
- 저장 데이터:
  title
  source
  published_at
  url
  summary
  raw_text(optional)
  collected_at

4. 정치인 매핑 엔진
- 기사 제목/본문에서 정치인 이름 탐지
- aliases 기준 매칭
- 동명이인은 직책/지역/정당 키워드로 보정
- 확신 낮은 매칭은 관리자 검수 대기

5. 키워드/테마 분류
- 초기 테마는 10~15개만
  부동산
  경제
  외교안보
  교육
  복지
  의료
  노동
  검찰/사법
  AI/기술
  저출산
  지역현안
- 기사별 테마 태깅
- 정치인별 자주 연결된 테마 집계

6. 지표 계산 엔진
- 정치인별 오늘 기사 수
- 정치인별 최근 7일 기사 수
- 전일 대비 변화율
- 7일 평균 대비 변화율
- 테마별 기사 수
- 정당별 기사 수
- 급증 기준은 “뉴스 언급량 기준”으로 명시

7. 관리자 검수 화면
- 수집 기사 확인
- 정치인 매핑 오류 수정
- 테마 태깅 수정
- 제외 기사 처리
- 문제 기사 숨김 처리
- 정정 요청 대비 로그 보관

8. 앱 홈 화면 MVP
- 오늘의 정치 흐름
- 뉴스 언급량 급증 인물
- 오늘 많이 언급된 인물
- 급증 테마
- 정당별 뉴스 흐름
- 사용자가 선택한 관심 인물 카드

9. 정치인 상세 화면
- 기본 정보
- 오늘 기사 수
- 최근 7일 흐름
- 연결 키워드
- 연결 테마
- 관련 기사 목록
- “모든 수치는 뉴스 데이터 기준” 표시

10. 관심 인물 기능
- 사용자가 관심 정치인 선택
- 홈 상단에 관심 인물 우선 배치
- 관심 인물의 오늘 변화 표시
- 푸시는 후순위

11. 디자인 방향
- 다크 모드 기본
- Bloomberg/TradingView 느낌
- 감정적 문구 금지
- 빨강/파랑 정치 색상 남발 금지
- 숫자, 카드, 차트, 근거 중심
- “좋다/나쁘다” 표현 금지

12. 법적/운영 안전장치
- 모든 지표에 기준 표시:
  “뉴스 언급량 기준”
  “수집 기사 기준”
  “자동 집계”
- 기사 원문 링크 제공
- AI 요약 사용 시 표시
- 정정 요청 메뉴 준비
- 선거기간 운영 정책은 별도 문서화

13. 1차 출시 기준
- 댓글 없음
- 토론 없음
- 예측 없음
- 평가 없음
- 뉴스 기반 지표만 제공
- 관심 인물 기반 홈 구성 가능
- 정치인 상세에서 근거 기사 확인 가능

14. 이후 확장
- 광역/기초 정치인 확대
- 정책시장 고도화
- 지역시장 추가
- 정당 상세 페이지
- 푸시 알림
- 장기 차트
- 언론 흐름 페이지
- 영향력 인물 시장
