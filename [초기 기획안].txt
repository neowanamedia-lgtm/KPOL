[초기 기획안]
프로젝트명(가칭): 대한민국 정치 흐름 터미널

1. 서비스 정의
- 대한민국 정치인을 주식시장처럼 분류하고, 뉴스와 공공 데이터를 기반으로 흐름을 보여주는 정치 데이터 인터페이스.
- 커뮤니티, 토론, 댓글, 사용자 의견 개진 기능은 두지 않는다.
- 운영자는 정치적 판단을 말하지 않는다.
- 서비스는 오직 데이터, 변화, 근거, 시계열 흐름만 보여준다.

2. 핵심 철학
- 우리는 주장하지 않는다.
- 우리는 평가 글을 쓰지 않는다.
- 우리는 의견을 받지 않는다.
- 우리는 대한민국 정치의 흐름을 보여준다.
- 모든 변화에는 근거가 있어야 한다.
- 근거는 뉴스, 공공 데이터, 공식 기록에 한정한다.

3. 서비스 포지션
- 정치 커뮤니티가 아니다.
- 정치 뉴스 사이트가 아니다.
- 선거 예측 도박 시장이 아니다.
- 정치인 팬덤 플랫폼이 아니다.
- 대한민국 정치 흐름을 관측하는 데이터 터미널이다.

4. 대상 범위
초기 대상:
- 국회의원
- 주요 정당 지도부
- 광역단체장
- 주요 차기 정치 후보군

확장 대상:
- 기초단체장
- 시·도 의원
- 기초의원
- 교육감
- 교육위원/교육행정 인물
- 주요 공공정책 관련 인물

5. 시장 분류 구조
주식시장 메타포를 적용한다.

예시:
- 중앙정치시장: 국회의원, 정당 지도부, 대권 후보군
- 광역정치시장: 광역단체장, 시·도지사, 광역의회 주요 인물
- 기초정치시장: 시장, 군수, 구청장, 기초의원
- 교육행정시장: 교육감, 교육 정책 관련 인물
- 정책섹터: 부동산, 복지, 교육, 경제, 외교안보, 노동, 환경 등
- 테마주: 특정 이슈와 강하게 연결된 정치인 그룹
- 대형주: 전국 인지도와 기사량이 높은 정치인
- 중소형주: 지역 기반 정치인

6. 핵심 화면
1) 메인 터미널
- 오늘의 정치 흐름
- 급상승 정치인
- 급하락 정치인
- 관심도 증가 인물
- 논란 기사 비중 증가 인물
- 정책 키워드 상승
- 시장별 랭킹

2) 정치인 상세 페이지
- 이름, 소속, 직책, 지역
- 최근 뉴스 흐름
- 일간/주간/월간 지수 변화
- 기사량 변화
- 정책 키워드
- 주요 이슈 타임라인
- 지수 변동 근거 뉴스
- 관련 공공 데이터

3) 시장/섹터 페이지
- 중앙정치시장
- 광역정치시장
- 기초정치시장
- 교육행정시장
- 정책 섹터별 흐름
- 테마별 정치인 묶음

4) 근거 페이지
- 지수 변동에 사용된 뉴스 목록
- 기사 날짜
- 언론사
- 원문 링크
- 자동 요약 여부 표시
- 지수 반영 항목 표시

7. 랭킹 시스템
단일 인기 순위는 지양한다.
다축 랭킹으로 설계한다.

예시 랭킹:
- 오늘의 관심도 상승
- 주간 언급량 상승
- 정책 영향력 상승
- 논란 기사 비중 상승
- 지역 정치 관심도 상승
- 장기 신뢰 흐름 상승
- 공공 활동 지수
- 뉴스 노출 지수
- 정책 연결 지수

주의:
- “최고의 정치인”, “최악의 정치인” 같은 표현 금지.
- “좋다/나쁘다” 식의 판단 표현 금지.
- “상승/하락/증가/감소/변동” 중심으로 표현한다.

8. 데이터 원칙
사용 가능한 데이터:
- 언론 뉴스
- 국회 공개 데이터
- 선관위 공개 데이터
- 지자체 공개 자료
- 공식 보도자료
- 공식 SNS/홈페이지 발언
- 판결문, 수사기관 공식 발표 등 공적 문서

사용하지 않는 데이터:
- 사용자 댓글
- 사용자 제보
- 익명 주장
- 커뮤니티 게시글
- 확인되지 않은 루머
- 출처 불명 자료

9. 사용자 참여 방식
허용:
- 시장/섹터 보기
- 정치인 검색
- 랭킹 확인
- 근거 뉴스 확인
- 관심 정치인 북마크
- 알림 설정

불허:
- 댓글
- 토론
- 평가글 작성
- 별점 평가
- 좋아요/싫어요
- 사용자 제보
- 정치인 비난/응원 게시물
- 커뮤니티 기능

10. 업데이트 주기
- 실시간 업데이트는 초기에는 하지 않는다.
- 하루 1회 또는 이틀 1회 업데이트를 기본으로 한다.
- 매일 새벽 배치 처리 방식 권장.

일일 처리 흐름:
1. 뉴스 수집
2. 정치인 이름/직책/지역 매핑
3. 중복 기사 제거
4. 기사 분류
5. 키워드 추출
6. 정치인별 흐름 계산
7. 시장/섹터별 지수 갱신
8. 랭킹 업데이트
9. 근거 뉴스 연결

11. 기술 엔진
핵심은 정치 데이터 엔진이다.

필요 모듈:
- 정치인 DB
- 뉴스 수집기
- 뉴스 중복 제거기
- 정치인 매핑 엔진
- 기사 분류 엔진
- 키워드 추출 엔진
- 지수 계산 엔진
- 근거 링크 저장 시스템
- 랭킹 생성 시스템
- 관리자 검수 도구

12. AI 역할
AI는 의견을 내지 않는다.
AI는 판단자가 아니라 분류자/정리자다.

AI 역할:
- 기사 요약
- 정치인 언급 추출
- 사건 키워드 분류
- 정책 분야 태깅
- 중복 기사 묶기
- 지수 변동 사유 요약
- 근거 뉴스 연결

AI 금지 역할:
- 정치적 논평
- 특정 정치인 평가문 작성
- 지지/비판 유도
- 선거 결과 단정
- 가치 판단 문장 생성

13. 법적/운영 리스크 대응
핵심 리스크:
- 명예훼손
- 선거법
- 정치적 편향 논란
- 허위사실 유포
- 뉴스 저작권
- 초상권/퍼블리시티권

대응 원칙:
- 사용자 생성 의견 차단
- 모든 지수 근거 공개
- 기사 원문 링크 제공
- AI 요약 표시
- 자동 산정 방식 명시
- 산식 공개
- 정정 요청 프로세스 마련
- 선거 기간 특별 운영 정책 마련

14. UI/UX 방향
- 금융 터미널 감성
- 블룸버그/트레이딩뷰/HTS 느낌
- 다크 모드 중심
- 숫자, 차트, 랭킹, 시계열 중심
- 감정적 문구 배제
- 자극적 제목 배제
- 정치 유튜브/커뮤니티 톤 금지

15. 초기 MVP 범위
초기에는 범위를 좁힌다.

MVP 추천:
- 국회의원 + 광역단체장 중심
- 주요 뉴스 출처 몇 곳부터 시작
- 하루 단위 업데이트
- 정치인별 뉴스량/언급량/키워드 흐름
- 시장별 랭킹
- 근거 뉴스 연결
- 댓글/토론/제보 기능 없음

16. 장기 확장
- 기초단체장 확대
- 시·도 의원 확대
- 기초의원 확대
- 교육감/교육행정시장 추가
- 정책 섹터 고도화
- 지역별 정치 흐름 지도
- 장기 시계열 아카이브
- 선거 기간 별도 흐름판
- 정치인 생애 흐름 차트

17. 핵심 문장 후보
- 대한민국 정치 흐름 터미널
- 우리는 의견을 만들지 않습니다. 대한민국 정치의 흐름을 보여줍니다.
- 정치를 말하지 않고, 흐름으로 보여줍니다.
- 뉴스와 공공 데이터로 보는 대한민국 정치 지형.
- 정치 커뮤니티가 아닌 정치 데이터 인터페이스.
